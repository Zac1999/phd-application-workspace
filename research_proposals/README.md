# Research Proposal 工作区

## 目录结构

```
research_proposals/
├── README.md                    # 本文件
├── CONSISTENCY_CHECKLIST.md     # RP与SOP一致性审核清单
├── drafts/                      # RP草稿版本
│   ├── v1.0_XJTLU_YYYYMMDD/   # 按学校和版本组织
│   └── v2.0_XJTLU_YYYYMMDD/
├── references/                  # 参考文献、相关论文
│   ├── key_papers/
│   └── citations/
└── reviewed/                    # 审核通过的最终版本
    └── final/
```

## 工作流程

1. **创建草稿** → 放入 `drafts/` 目录
2. **内容审核** → 对照 `CONSISTENCY_CHECKLIST.md` 检查与SOP一致性
3. **修改完善** → 根据审核结果迭代
4. **最终定稿** → 移至 `reviewed/final/`

## 关键约束

- **必须与 SOP 保持一致**：研究方向、经历描述、技术术语
- **必须匹配导师方向**：基于 `/user_data/verified/wen_liu_professor.json`
- **必须基于已验证信息**：参考 `/user_data/verified/yuhang_liu_complete.json`
